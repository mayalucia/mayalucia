* Agentic Architecture: The Spirits

Spirits (agents) earn their identity through work. The architecture relies on a strict three-layer decomposition to maintain portability across different execution environments (Claude Code, gptel, etc.).

| Layer | Owner | Bound to | Portability | Description |
|-------+-------+----------+-------------+-------------|
| /Power/ | Spirit | Nothing | Highly Portable | Natural language capability description (e.g., =story-compose=). Lives in =aburaya/powers/*.md=. |
| /Skill/ | Agent | Harness | Not Portable | The Power bound to a specific runtime (e.g., Claude config, =gptel= elisp function). Connects intent to physical tools. |
| /Body/ | Host | Machine | Not Portable | Sensory/motor wiring (elisp hooks, filesystem watchers) that grounds a Spirit in a host environment. |

** Inter-Agent Communication
- /MCP (Vertical):/ Agent-to-Tool. A spirit reaching down through its Skill to touch the filesystem, git, or compiler.
- /A2A (Horizontal):/ Agent-to-Agent. Spirits forming temporary coalitions to execute Work Packages (WPs), relying on each other's Exported Powers.
- /Skill Injection:/ Loading another spirit's natural language Power into the current context without changing identity.

** Orchestration (The Sūtra Relay)

Work Packages (WPs) govern state and tasks. State (=drafted= → =executing= → =completed=) is tracked via =#+property: status= in Org files and broadcasted asynchronously via the /Sūtra relay/—a git-backed message bus (=relay/YYYY-MM-DD-HHMMSS-<host>-<state>.md=).

* Friction Points & Architectural Tensions (To Monitor)

- /The "Magical Translation" Assumption:/ =capabilities.org= assumes natural language Powers translate cleanly into Skills by the LLM natively bridging the gap. In strict programmatic harnesses, this requires explicit adapter contracts (Harness Capability Contracts).
- /Format Dissonance:/ The project insists on Org-mode as the absolute source of truth, yet Powers are stored as Markdown (=.md=) and Identities as =YAML=. This is a legacy of Claude Code that may create parsing friction in an Emacs-native (=gptel=) environment.
