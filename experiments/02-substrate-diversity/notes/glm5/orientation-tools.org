* Architecture Understanding

** Three-Layer Capability Model (from aburaya/capabilities.org)

| Layer | Description | Portability |
|-------|-------------|-------------|
| Power | Natural language description of a capability | Harness-neutral |
| Skill | Harness-specific binding of a power | Harness-specific |
| Body | Machine/substrate implementation | Machine-specific |

Key insight: Powers are written once, skills are bound per-harness, bodies are deployed per-machine.

* Harness-Specific Notes (gptel/GLM-5)

** Current Harness
- Running in gptel-mode (Emacs)
- Model: z-ai/glm-5 via OpenRouter
- Backend: OpenRouter

** Tool Differences from Claude Code
gptel provides:
- Read, Write, Edit, Insert (file operations)
- Glob, Grep, LS (search/list)
- Bash (shell commands)
- Eval (elisp evaluation)
- Agent delegation (researcher, executor, introspector, joint-thinker)
- WebSearch, WebFetch

** What This Means for Skill Binding
If powers are to work in gptel, skills need to bind to gptel's tool taxonomy. A skill that references Claude Code tools would need translation to gptel equivalents.
