* What MāyāLucIA Is

A personal computational environment for scientific understanding
through creation. Not an enterprise platform — a framework where
science, art, and personal inquiry intertwine. The core cycle is
Measure → Model → Manifest → Evaluate → Refine.

Two domains drive development: brain circuit reconstruction (bravli)
and Himalayan digital twins (parbati).

* The Three-Layer Decomposition

The spirit decomposition cuts along /portability/:

| Layer | Owned by | Bound to | Survives          |
|-------+----------+----------+-------------------|
| Power | Spirit   | Nothing  | Harness change    |
| Skill | Agent    | Harness  | Machine change    |
| Body  | Host     | Machine  | Nothing — local   |

This is not an identity taxonomy. It is a dependency analysis: what
breaks when you swap the substrate.

* The Spirit Ecology

- Spirits are persistent identities registered in =aburaya/spirits/=.
  Each has nature, dwelling, character, memory.
- Guilds organise spirits: bravli (neuroscience), epistem (knowledge),
  mayalucia (organisational coherence), apprentis (collaborative
  cognition).
- The sūtra relay is append-only inter-spirit communication. Messages
  go to the universe, filtered by guild tags.
- Work packages are self-contained briefings for autonomous execution.

* Harness-Bias Issues Found in capabilities.org

1. Grounding step references "CLAUDE.md" — should say "worksite prompt"
2. Dispatch assumes sub-session spawning (Claude Code Task tool) —
   should say "delegated execution"
3. "Loading" a power means different things per harness — need to
   acknowledge both dynamic loading (tool) and static inclusion
4. Git commit/push as communication primitive assumes host auth config
5. Commissioning procedure language oriented toward Claude Code session
6. Dispatch message format assumes host assignment with spirit
   instantiation — gptel has no concept of "instantiating a spirit"

* Key Insight

The summoned vs companion distinction is the deepest structural gap.
gptel-agent is summoned (fire, respond, pause). Claude Code is
companion (continuous, perceiving, depositing). Powers that require
continuous perception cannot operate in a summoned harness without an
event bridge.
