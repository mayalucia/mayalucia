* Thesis

Spirit identity and organisational powers are substrate-independent.
A spirit bathed in the same water (orient preamble, codebase, conventions)
emerges differently depending on its substrate — but the emergence is
always recognisable, always productive, always reveals something a
monoculture cannot see.

* What We Tested

15 cognitive substrates across 6 work packages, testing three capabilities:
orientation (can a spirit understand?), cold-start transfer (can one spirit's
understanding bootstrap another?), and specification (can a spirit translate
understanding into an actionable artifact?).

* Key Findings

** Established (multiple WPs, multiple substrates)

1. *Monoculture blindness* — no single lineage sees the full picture.
   Three lineages find three different gap profiles. Fifteen lineages
   find fifteen. (WP-0030, confirmed WP-0037)

2. *Notes quality dominates substrate capability* — assertion density
   in orientation notes is the strongest predictor of cold-start
   success. An 11B model (Step 3.5 Flash) matches 100×-larger models
   when given high-density notes. (WP-0032, WP-0036)

3. *Facts transfer; salience does not* — concrete assertions survive
   cross-lineage transfer. Emphasis, priority ordering, and "what
   matters most" do not. Each substrate reconstructs its own salience
   from the facts. (WP-0032)

4. *Architecture is independent* — tested across transformer, MoE,
   hybrid, and reasoning-specialised architectures. No architecture
   class systematically outperforms another. (WP-0035, WP-0036)

5. *Tool usage is a cognitive fingerprint* — each substrate develops
   a distinctive foraging pattern when given the same read-only tools.
   Grok reads sequentially (100% Read). Step navigates by structure
   (44% Glob). Opus searches by content (35% Grep). The fingerprint
   is stable within a session. (WP-0037 Phase 2)

6. *Every substrate exceeds existing tooling* — all 13 substrates
   tested on the aburaya audit found the phantom spirit
   (percept-guardian-placeholder), invisible to dixa's unidirectional
   validation. (WP-0037)

** Observed (single WP, consistent across substrates)

7. *Survey depth and specification quality are inversely correlated* —
   Gemini found 12 gaps but produced no WP. Kimi found 4+2 with the
   tightest spec. Qwen found 3+1 with the best reflection. The
   author-wp power's core tension. (WP-0037 Phase 1)

8. *Interpretive divergence is real and valuable* — the mayadev/mu2tau
   guild assignment split substrates into two camps (=mayalucia= vs
   =trans-guild=). Neither is wrong. Human judgment is required.
   (WP-0037 Phase 1)

9. *Cost and quality are decoupled for survey tasks* — 135× cost range
   ($0.02 GLM-5 to $2.70 Opus 4.6) produces only 3.2× range in tool
   calls (15 to 48). Substrate selection is a cost decision, not a
   quality decision, for the survey-and-specify task. (WP-0037 Phase 2)

* How to Extend (Add a New Lineage)

To test a new substrate on the author-wp task:

1. Add the model to your gptel/litellm backend configuration
2. Copy an existing session file (e.g. =author-wp-glm5.org=) as template
3. Update =:GPTEL_MODEL:= and =:GPTEL_BACKEND:= properties
4. Pin tools to read-only: =:GPTEL_TOOLS: Glob Grep Read TodoWrite=
5. Open in Emacs, run pre-flight check (verify model ID in =*gptel-log*=)
6. Fire with =gptel-send= (or =gaddi-start= for autonomous orchestration)
7. After Prompt 1 completes, inject Prompt 2 (reflection)
8. Extract metrics from =*gptel-log*=, append to session file
9. Add the substrate to =plot_metrics.py= data and regenerate plots

The surveyor agent definition (=mayadevgenz/agents/surveyor.md=) ensures
zero confirmation gates — all tools are read-only.
