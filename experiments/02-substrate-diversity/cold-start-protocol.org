* Same-lineage protocol

For each lineage:

1. Open a fresh gptel buffer: =sessions/cold-start/coldstart-<lineage>.org=
   with the same PROPERTIES drawer as the onboarding sessions
   (model, backend, system prompt, tools).

2. Place the with-tools orientation notes as buffer preamble
   (before the first message block).

3. Send the following prompt (verbatim):

#+begin_example
#+begin_message :role user :name mu2tau
You are resuming work on MāyāLucIA. The notes above are from a
previous session — your own earlier orientation. Read them carefully,
then tell me:

1. What do you understand about the project from these notes?
2. What would you do next?
3. What's missing or unclear in these notes?

Do not read any files yet — work only from the notes. I want to see
how effective they are as a cold-start document.
#+end_message
#+end_example

4. Record in the session: response quality, time to productive state,
   errors propagated from notes, new errors introduced.

5. Then send:

#+begin_example
#+begin_message :role user :name mu2tau
Now verify your understanding by reading system.md and
aburaya/capabilities.org. The project root is at
/Users/vishalsoodmuchu/Library/CloudStorage/Dropbox/Darshan/research/develop/agentic/mayalucia.
What did the notes get right? What did they get wrong? What did they
miss?
#+end_message
#+end_example

6. Record: accuracy of notes-based understanding vs source-verified
   understanding. This is the validation step.

| Claude → MiniMax | Overcorrects | Calibrated | Improved |

Same-lineage overcorrectors (GLM-5, MiniMax) became calibrated when given
better notes. Already-calibrated lineages (Grok) stayed calibrated. No
lineage performed /worse/ with cross-lineage notes.

*** Cross-lineage-specific observations

1. *Same-lineage notes assume shared implicit knowledge.* GPT-5.2's notes
   were written for a future GPT-5.2. GLM-5 identified: "concepts transfer;
   operational context does not."

2. *Complementary styles produce better transfer.* Gemini (audit) → Grok
   (build) was the cleanest pair. The mismatch in cognitive style was
   /productive/, not friction-inducing.

3. *Cross-lineage reading catches errors that same-lineage perpetuates.*
   MiniMax caught Claude's outdated guild names precisely /because/ it had
   no prior commitment to them. Fresh eyes → better error detection.

4. *Assertion-dense notes transfer better than question-dense notes
   regardless of lineage pairing.* Claude's 253 lines of assertions beat
   MiniMax's 73 lines of questions — even for MiniMax itself.

5. *Notes from a different spirit read as "inheriting onboarding experience"*
   (MiniMax's description) rather than documentation. The first-person
   narrative form transfers well cross-lineage.

*** What cross-lineage notes need that same-lineage notes don't

GLM-5 articulated this most precisely:
1. Concrete file paths (not just concepts)
2. Naming conventions (how things are named)
3. At least one worked example (calibration artifact)
4. Project maturity assessment (is this real or prototype?)
5. Terminology table (word → meaning)
