* Context

The charter's Phase 1 (AI/LLM evaluation) and Phase 3 (finance domain)
are decomposed into four executable WPs. WP-0021 (finance) is pulled
forward from Phase 3 because it serves a dual purpose: DMT domain
demonstration *and* the human's entry into quantitative finance.

All four WPs execute inside =modules/dmt-eval/=. The executor for all
is =mayadev= (the human + agent pair working in the dmt-eval module).

* Parallelism

WP-0018, WP-0019, and WP-0021 can begin in parallel — each has
substantial independent work before needing CLI integration. WP-0020
blocks on both 0018 and 0019.

Recommended execution order for a single executor:

1. *WP-0018* first — provides the CLI scaffold that 0019 and 0021
   plug into
2. *WP-0021* second — the learning priority; builds intuition for
   quantitative finance through construction
3. *WP-0019* third — LLM evaluation, builds on CLI
4. *WP-0020* last — MCP server, wraps everything
