* Context

The charter's Phase 1 (AI/LLM evaluation) and Phase 3 (finance domain)
are decomposed into four executable WPs. WP-0025 (finance) is pulled
forward from Phase 3 because it serves a dual purpose: DMT domain
demonstration *and* the human's entry into quantitative finance.

All four WPs execute inside =modules/dmt-eval/=. The executor for all
is =dmt-eval-guardian= — the validation framework's guardian spirit,
working in its home module.

* Parallelism

WP-0022, WP-0023, and WP-0025 can begin in parallel — each has
substantial independent work before needing CLI integration. WP-0024
blocks on both 0022 and 0023.

Recommended execution order for a single executor:

1. *WP-0022* first — provides the CLI scaffold that 0023 and 0025
   plug into
2. *WP-0025* second — the learning priority; builds intuition for
   quantitative finance through construction
3. *WP-0023* third — LLM evaluation, builds on CLI
4. *WP-0024* last — MCP server, wraps everything
