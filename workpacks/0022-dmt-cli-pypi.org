* Context and Motivation

DMT-Eval has a working Python API: =dmt.evaluate()= produces LabReports
from (models, observations, scenario) tuples. Two complete scenarios
(weather, drug efficacy), a Brain-Score domain adapter, and a Streamlit
bench service exist. But there is no CLI — every evaluation requires
writing a Python script.

The charter's Level 0 API promises:

#+begin_example
dmt eval --model openai/gpt-4o --dataset evals/qa.csv --metrics accuracy,latency
dmt compare --models model_a,model_b --dataset data.csv --output ./report
dmt report ./dmt_report --format html
#+end_example

This is the difference between "a library you can import" and "a tool
you can use." The CLI makes DMT accessible from shell scripts, CI
pipelines, agent harnesses, and =Makefile= targets.

PyPI release makes DMT installable with =pip install dmt-eval=. Without
it, adoption requires cloning a repo. The =pyproject.toml= already
exists and is nearly ready — what's missing is the CLI entry point,
version management, and a release workflow.

* Specification

** Part 1: CLI (=src/dmt/cli.py=)

Three commands via =typer=:

*** =dmt eval=

Run a single model against a dataset. Minimal invocation:

#+begin_src shell
dmt eval --scenario weather
#+end_src

This uses built-in synthetic data and all three weather models.
Produces a LabReport in =./dmt_report/=.

Full invocation:

#+begin_src shell
dmt eval --scenario weather \
         --data observations.csv \
         --model "my_model:path/to/module.py" \
         --output ./results \
         --title "Q1 Weather Forecast Validation"
#+end_src

*** =dmt compare=

Compare multiple models (the tournament use case):

#+begin_src shell
dmt compare --scenario drug_efficacy \
            --models linear,sigmoid,calibrated \
            --reference linear \
            --output ./comparison
#+end_src

*** =dmt report=

Re-render an existing report directory in a different format:

#+begin_src shell
dmt report ./dmt_report --format html
dmt report ./dmt_report --format pdf   # Phase 2
#+end_src

*** =dmt list=

List available scenarios, models, and metrics:

#+begin_src shell
dmt list scenarios
dmt list metrics
#+end_src
