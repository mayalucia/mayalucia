mu2tau noticed that the project's 62+ Org files had no systematic
navigation beyond grep and memory. The WP-0042 convention dispatch
experiment had already identified discoverability — not availability —

and framing the tagging pass as a substrate experiment: same documents,
different substrates, different tag choices.

* Context

The project has 62+ Org files across =workpacks/=, =develop/=, and
=aburaya/=. Navigation relies on memory, filename convention, and
grep. Org's native tag system (=#+filetags:= and headline tags) is
unused. As the project grows, discoverability — not availability — is
the bottleneck (WP-0042 insight).

Tagging is also a substrate experiment: reading a document, choosing
the right tags, and extending the catalog requires /judgment/, not
just comprehension. Different substrates will see different concerns
in the same document. The tag catalog's evolution across substrates is
itself data for the apprentis guild.

* Specification

** 1. Tagging protocol

For each Org file, a two-phase process:

*Phase A — Headlines (bottom-up)*

1. Read the file's headlines (all levels, down to the finest)
2. Tag each headline with =:tag1:tag2:= drawn from the catalog
3. Produce a brief summary of the file (constrained by context)
4. If no existing tag fits a headline, propose a new tag with a
   one-line definition. Record the addition.

*Phase B — Filetags (top-down)*

5. From the set of headline tags and the summary, derive =#+filetags:=
6. Add the =#+filetags:= line after existing =#+property:= lines

This two-phase design keeps context manageable — large files need not
be sent whole to the substrate. The headline scan (Phase A) is the
expensive read; filetag derivation (Phase B) works from the Phase A
output.

Going forward, when new content is written (new files, new headlines,
or edits to existing headlines), the author tags that content
incrementally. The full-file pass is a one-time cost.

#+begin_insight
The two-phase design mirrors the sutradhar companion's
perception-cognition loop: Phase A is /attend/ (scan the document, tag
what you see — like =attend-working-context=), Phase B is /synthesise/
(derive the whole from the parts — like =hold-the-thread=). If run as
a substrate experiment, the Phase A → Phase B transition is where
substrate character will show most clearly — same headlines, different
filetags.
#+end_insight
