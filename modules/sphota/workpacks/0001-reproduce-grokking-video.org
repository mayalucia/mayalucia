* CONTEXT
The Welch Labs video, "The Most Complex Model We Actually Understand," provides a compelling visual and narrative account of the "grokking" phenomenon discovered by OpenAI. It serves as an ideal, self-contained entry point into the core concerns of the =percept= guild: understanding how and when a neural network transitions from memorization to generalization. This work package commissions the creation of a complete, end-to-end computational framework to reproduce every key visualization in that video, structured as a literate program in accordance with MāyāLucIA's development philosophy. This WP constitutes **Phase 1** of the broader Rosetta experiment ([[../experiments/grokking-rosetta.org][grokking-rosetta.org]]), focusing solely on the **JAX/Equinox** implementation. Future phases will address the TinyGrad and Julia paradigms.

* SPECIFICATION
The assigned spirit shall create a literate program that implements a complete computational metrology framework. This framework must be capable of generating the necessary data and visualizations to recreate the Welch Labs video. The work will be divided into three main components, tangled from a single org-mode source of truth.

**1. The Simulation Core**
The framework must implement the core experiment.
- **Data Generator:** A function to generate the modular arithmetic dataset for any given modulus, with a configurable train/test split. Must default to a ~30-40% training split, which is critical for inducing grokking.
- **Model:** A 1-layer Transformer architecture, precisely matching the one used in the grokking papers, implemented in JAX/Equinox.
- **Instrumented Trainer:** A training loop that uses an **AdamW optimizer with significant weight decay** (e.g., 1.0), which is essential for grokking, and at regular intervals, saves:
    1. Full model parameter snapshots.
    2. Train/Test loss and accuracy.
    3. Full activation tensors for a grid of all possible inputs.

**2. The Analysis & Visualization Core**
The framework must implement the "measurement" tools to analyze the simulation's output.
- **Metric Plotter:** A function to generate the 2D plot of train/test loss/accuracy over time.
- **Activation Surface Renderer:** A function to render the 3D surface plot of a single neuron's activation across the input grid.
- **Fourier Module:** A module to compute and visualize the 2D Discrete Fourier Transform of an activation surface.
- **Mechanistic Intervention Engine:** A module to calculate the "excluded loss" by running a forward pass while programmatically zeroing out specific frequency components in activation space.

**3. The Narrative & Rendering Engine (Stretch Goal)**
The framework must include a top-level script to assemble the analyses into a final video artifact.
- **Frame Renderer:** A function that, for any given training step `t`, can generate all necessary plots for that point in time.
- **Video Assembler:** A script that iterates through the saved training snapshots, calls the frame renderer for each step, and uses `ffmpeg` to stitch the resulting image files into an MP4 video.
